# Infrastructure (Terraform)

Everything AWS lives here — no console clicking. `terraform apply` goes from an
empty account to a running, load-balanced API.

```
Client ──HTTPS──▶ ALB (ACM cert) ──▶ EC2: Docker(api + postgres) ──▶ CloudWatch logs
```

| File | Contains |
| --- | --- |
| `vpc.tf` | VPC, 2 public subnets (ALB needs 2 AZs), IGW, routes |
| `security.tf` | ALB SG (80/443 from world), app SG (8000 from ALB only, optional SSH) |
| `ec2.tf` | Amazon Linux 2023 instance + cloud-init that runs the Docker stack |
| `user_data.sh.tpl` | Installs Docker, writes the prod compose file, starts it |
| `alb.tf` | ALB, target group with `/health` checks, HTTP/HTTPS listeners |
| `acm.tf` | TLS cert + DNS validation (only when `domain_name` is set) |
| `cloudwatch.tf` | Log group — the API's JSON logs ship via Docker's awslogs driver |
| `iam.tf` | Instance role: CloudWatch Logs write access, nothing more |

## Deploy

Prerequisites: an AWS account, credentials configured (`aws configure` or env
vars), and the Docker image published to GHCR (CI does this on every push to
`main` — make the package public in GitHub once, under Package settings).

```bash
cd terraform
cp terraform.tfvars.example terraform.tfvars   # edit: at minimum app_image
terraform init
terraform plan
terraform apply
# outputs api_url -> open <api_url>/docs
```

Without a `domain_name` the ALB serves plain HTTP (no cert without a domain).
With `domain_name` + `route53_zone_id` set you get ACM + HTTPS + redirect.

## Tear down

```bash
terraform destroy
```

Stops all billing. State is local (fine for a single-person portfolio project;
a team would use an S3 backend with state locking).

## Decisions

- **Dockerized Postgres on the instance, not RDS.** RDS is the production
  answer (backups, failover, patching) but costs ~€15+/mo extra. For a
  single-instance portfolio demo, Postgres in Docker with a named volume is
  honest about its trade-offs — and this file documents them.
- **Public subnets only, no NAT gateway.** A NAT gateway (~€32/mo) would
  triple the bill. The instance is protected by its security group instead:
  port 8000 accepts traffic from the ALB's security group only.
- **Secrets** (JWT secret, DB password) are generated by `random_password` at
  apply time and live only in user_data + Terraform state — never in git. At
  bigger scale: AWS Secrets Manager.
- **IMDSv2 enforced** (`http_tokens = "required"`), minimal IAM (logs only).
